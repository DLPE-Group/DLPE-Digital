# S6 — Billing & Subscriptions (thin) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Every tenant has a subscription to a plan; the app can answer "is this tenant active?" and "does its plan grant feature X / allow N of Y?"; a platform admin can see/change a tenant's plan — all behind a pluggable, simulated billing provider.

**Architecture:** `plans → entitlements` model (`Plan` + per-tenant `Subscription`), a `BillingProvider` seam (`SimulatedBillingProvider` default, `StripeBillingProvider` stub), entitlement helpers, one real gate (`maxUsers` on user creation), provisioning assigns a default subscription, and control-plane UI to view/change a tenant's plan.

**Tech Stack:** Express + Prisma 6 + Postgres, Vite/React, Vitest API + Playwright.

## Global Constraints
- Builds on S0+S1+S3 (all merged to main). Branch: `feature/s6-billing` (off main).
- **Billing stays pluggable** (roadmap tenet): a plan's grants live in `Plan.entitlements` JSON (`{ features: string[], limits: Record<string,number> }`); changing the pricing model = editing plans/entitlements, NOT a schema migration.
- **`Plan` is platform-level** (no `tenantId`, no RLS) — like `Blueprint`. **`Subscription` IS tenant-scoped** (has `tenantId`, gets RLS like the other 26 tenant models). Provisioning/platform ops use the owner `prisma` (bypass RLS).
- **External billing stays SIMULATED** (like Nango/email). `SimulatedBillingProvider` makes no network calls. `StripeBillingProvider.* ` throws `Error('Stripe billing is deferred to a future spec')`.
- Mirror existing seams: `server/src/auth/AuthProvider.ts`, `server/src/domain/provisioning/target.ts`.
- Subscription statuses: `TRIALING | ACTIVE | PAST_DUE | CANCELED`. "Billable-active" = `TRIALING|ACTIVE`.
- Plans: `starter` (maxUsers 10, features [reports]), `pro` (maxUsers 50, features [reports, api_access]), `enterprise` (unlimited users, all features). Demo tenant → `enterprise`/ACTIVE.
- Migrations: hand-author + `prisma migrate deploy` (never `migrate dev`/`reset`); test DB via `node tests/prepare-db.mjs`. Adding RLS to the new `Subscription` table must follow the S0 pattern (ENABLE+FORCE+policy `tenantId = current_setting('app.tenant', true)` + grant to il_app).
- Commit trailer ends with `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.

### Task 2: BillingProvider seam

**Files:** Create `server/src/domain/billing/provider.ts`; Test `tests/api/billing.test.mjs`.

**Interfaces:**
- Consumes: `prisma`, `Plan`/`Subscription`.

  - `interface BillingProvider { createSubscription(a:{tenantId:string;planKey:string;seats?:number;status?:string}): Promise<SubscriptionState>; changePlan(a:{tenantId:string;planKey:string}): Promise<SubscriptionState>; cancel(a:{tenantId:string}): Promise<SubscriptionState>; }`

  - `class StripeBillingProvider implements BillingProvider` — all methods throw `Error('Stripe billing is deferred to a future spec')`.
  - `export const billingProvider: BillingProvider = new SimulatedBillingProvider();`
